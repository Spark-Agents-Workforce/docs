# connecting your data

> how to plug in your tools and start the stream flowing.

spark doesn't generate data. your team does. spark watches.

connecting a tool means its events flow into the activity stream in real-time. every commit, every PR, every slack message, every task completion — as it happens. the system groups those events into sessions, scores them, synthesizes them, and makes them queryable.

## what changes when you connect

connecting your first tool is the moment spark goes from empty to alive. here's what happens:

<Steps>
  <Step title="events start flowing" icon="activity">
    within seconds of connecting, events appear in the activity feed. you can see your team's work arriving in real-time.
  </Step>

  <Step title="sessions form" icon="layers">
    the system watches the event stream and groups related events into sessions. within a few hours of normal team activity, you'll see your first scored sessions.
  </Step>

  <Step title="agents activate" icon="eye">
    the built-in agents — impact scorer, synthesis generator, signal puller — start processing. sessions get scored. by end of day one, you have your first daily synthesis.
  </Step>

  <Step title="intelligence compounds" icon="trending-up">
    every day adds more data. scoring gets better. syntheses get richer. patterns emerge that weren't visible in any single tool. this is the compounding effect.
  </Step>
</Steps>

## layering tools

each tool you connect adds a new dimension. the power is in the combination.

<Note>
  **github alone:** you see coding sessions — commits, PRs, reviews. who wrote what code, when, how it was reviewed.

  **github + slack:** now sessions have context. that burst of commits happened right after an urgent slack thread. the impact scorer sees the connection and adjusts the importance score.

  **github + slack + asana:** the full picture. code sessions link to task completions. you can see the journey from "task created" to "PR merged" to "task done." gap detection works — the system notices when tasks are moving without corresponding code activity.

  **+ shopify:** ecommerce context. did that deploy correlate with an order drop? did the checkout fix actually increase conversion?
</Note>

the more tools you connect, the richer the sessions, the smarter the scoring, the more useful spark's answers become. each tool on its own is a partial view. together, they're the full picture.

## getting started

the fastest path to value:

1. **connect github.** this is the primary integration. most teams get their first scored sessions within hours.
2. **add slack** when you want context beyond code. the signal puller starts capturing non-code activity and the impact scorer gets richer context.
3. **add your task tracker** (asana or linear) to close the loop between planning and execution.
4. **add custom webhooks** for anything else — internal tools, CI/CD pipelines, or services spark doesn't natively support.

you don't need everything connected on day one. start with github and layer from there.
