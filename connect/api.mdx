# API & CLI

> query all spark data over HTTP or command line — seven read-only endpoints.

the spark API exposes seven read-only endpoints for querying your data. the CLI wraps every endpoint — same data, same params, terminal-native.

<Note>
  all query endpoints are read-only. the API cannot modify data or change configuration. event ingestion is handled separately via [integrations](/connect/integrations) and [webhooks](/connect/webhooks).
</Note>

## setup

### API authentication

1. generate an API key in **settings → API**
2. pass the key in every request:

```bash
Authorization: Bearer obs_your_api_key
```

**base URL:** `https://your-deployment.convex.site`

### CLI install

```bash
npm install -g @sparkagents/cli
```

configure your shell profile (`~/.zshrc`, `~/.bashrc`):

```bash
export OBSERVATORY_URL=https://your-deployment.convex.site
export OBSERVATORY_API_KEY=obs_your_api_key
```

the CLI outputs JSON by default and pipes cleanly to `jq`.

## GET /api/v1/team

team overview with rankings, impact scores, sparklines, and status.

| param    | type                          | default | description                  |
| -------- | ----------------------------- | ------- | ---------------------------- |
| `period` | `today` \| `week` \| `month` | `today` | time window for the overview |

<CodeGroup>
  ```bash CLI
  spark team
  spark team --period week
  spark team --period month
  ```

  ```bash curl
  curl https://your-deployment.convex.site/api/v1/team?period=week \
    -H "Authorization: Bearer obs_your_api_key"
  ```

## GET /api/v1/events

filtered event stream. all params are optional — omit to get the full stream.

| param    | type     | default | description                                        |
| -------- | -------- | ------- | -------------------------------------------------- |
| `actor`  | string   | —       | filter by actor identifier                         |
| `source` | string   | —       | filter by source tool (e.g. `github`, `slack`)     |
| `repo`   | string   | —       | filter by repository name                          |
| `type`   | string   | —       | filter by event type (e.g. `push`, `pull_request`) |
| `since`  | ISO 8601 | —       | return events after this timestamp                 |
| `until`  | ISO 8601 | —       | return events before this timestamp                |
| `limit`  | number   | `50`    | max results returned (max 200)                     |

<CodeGroup>
  ```bash CLI
  spark events
  spark events --actor sam
  spark events --actor sam --since friday
  spark events --source github --repo app-core
  spark events --type pull_request --limit 20
  ```

  ```bash curl
  curl "https://your-deployment.convex.site/api/v1/events?actor=sam&source=github&limit=10" \
    -H "Authorization: Bearer obs_your_api_key"
  ```
</CodeGroup>

<Tip>the CLI accepts natural-language time shortcuts for `--since` and `--until`: `today`, `yesterday`, `monday`, `friday`.</Tip>

```json response
{
  "events": [
    {
      "id": "evt_7hq4p",
```

## GET /api/v1/summary

AI-generated synthesis for a time period. returns the narrative summary with key themes and highlights.

| param    | type                              | default | description                 |
| -------- | --------------------------------- | ------- | --------------------------- |
| `date`   | `YYYY-MM-DD`                      | today   | target date for the summary |
| `period` | `daily` \| `weekly` \| `monthly` | `daily` | synthesis window            |

<CodeGroup>
  ```bash CLI
  spark summary
  spark summary --date 2026-04-07
  spark summary --period weekly
  spark summary --period monthly
  ```

  ```bash curl
  curl "https://your-deployment.convex.site/api/v1/summary?date=2026-04-07&period=daily" \
    -H "Authorization: Bearer obs_your_api_key"
  ```
</CodeGroup>

## GET /api/v1/health

system health across all subsystems. useful for monitoring and alerting.

<CodeGroup>
  ```bash CLI
  spark health
  ```

  ```bash curl
  curl https://your-deployment.convex.site/api/v1/health \
    -H "Authorization: Bearer obs_your_api_key"
  ```
</CodeGroup>

```json response
{
  "webhook": {
    "status": "healthy",
    "lastEventAt": 1712534400000,
    "eventsLast24h": 342
  },
  "scoring": {
    "status": "healthy",
    "unscoredSessions": 0,
    "lastScoredAt": 1712534200000
  },
  "synthesis": {
    "status": "healthy",
    "lastSynthesisAt": 1712520000000,
    "lastPeriod": "daily"
  },
  "overall": "healthy"
}
```

`overall` is `healthy` when all subsystems are healthy, `degraded` when one or more are behind, and `unhealthy` when a subsystem has stopped.

<Tip>the CLI exits with code 1 if any subsystem is unhealthy, making it suitable for monitoring scripts and CI checks.</Tip>

## pipe to jq

all CLI output is valid JSON. a few useful patterns:

<CodeGroup>
  ```bash team rankings
  spark team --period week | jq '.members[] | {actor, impact, status}'
  ```

  ```bash recent events
  spark events --actor maya --limit 5 | jq '.events[] | {type, timestamp, payload.title}'
  ```

  ```bash health check
  spark health | jq '.overall'
  ```

  ```bash unmapped work
  spark unmapped --since monday | jq '.sessions[] | {actor, summary, impact}'
  ```
</CodeGroup>
