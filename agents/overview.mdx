agents are where spark goes from "tool you use" to "system that works for you."

instead of you asking questions, agents watch your data streams and tell you when something matters. they're the intelligence layer that turns raw data into scored sessions, synthesized summaries, ingested knowledge, and actionable alerts.

## three capabilities

every agent is built from three composable capabilities:

<Steps>
  <Step title="watch" icon="radio">
    monitor a stream for events or patterns. every agent does this.
  </Step>
  <Step title="evaluate" icon="scan">
    AI decides what the event means. scores confidence. classifies. compares to what's known. most agents do this.
  </Step>
  <Step title="act" icon="zap">
    take an external action. send an alert. create a wiki page. route a message. score a session. some agents do this.
  </Step>
</Steps>

these are building blocks, not a rigid pipeline. some agents watch and evaluate — they're **analysts**. some watch, evaluate, and act — they're **operators**.

## agents are toggleable

every agent can be turned on or off. some ship on by default because they're core to the experience. others are available to enable when you need them.

<Tabs>
  <Tab title="on by default">
    - **impact scorer** — scores every session on impact, output, importance, quality
    - **synthesis generator** — writes daily and weekly narrative summaries
    - **signal puller** — monitors external tools for non-code events
    - **knowledge ingest** — processes new sources into wiki pages
  </Tab>
  <Tab title="available to enable">
    <CardGroup cols={2}>
      <Card title="PR quality monitor" icon="git-pull-request">
        flags risky merge patterns
      </Card>
      <Card title="slack urgency detector" icon="bell-ring">
        identifies escalations and urgent messages
      </Card>
      <Card title="deploy watcher" icon="rocket">
        correlates deploys with error signals
      </Card>
      <Card title="shopify health monitor" icon="shopping-bag">
        detects order anomalies
      </Card>
      <Card title="knowledge lint" icon="shield-check">
        health-checks the knowledge base for contradictions and gaps
      </Card>
    </CardGroup>
  </Tab>
</Tabs>

turn on what matters. turn off what's noise. each agent is a specialist — one clear purpose, one toggle.
